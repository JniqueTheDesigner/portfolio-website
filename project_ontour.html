OnTour: Money Saving Tool for Indie Artists

Overview

On Tour is a web based Fintech app that provides financial and career development guidance to Independent Artists. The UI design took about 3-4 weeks to complete, which includes forming Branding Guidelines, User Flows, Wireframes, Iconography and User Testing.

My Process

Branding Guidelines

Before designing it was important to form brand guidelines, so the UI had a premise to how it should connect or be perceived by potential users. In this stage the logo, brand colors, typography, and guiding principles were created.

User Flow

On Tour is meant to be tailored for an artist and their career individually, so JTBD had to have features that allowed users to personalize their account based on their music career. One of the main user flowâ€™s being Onboarding for a New User, which then serves as a starting point for creating an account, and user financial dashboard.

Low Fid Wireframes

Low fid wireframes for user flow #1 : Onboarding (New User) was designed first, to give an idea to how the layout of UI elements need to be arranged so users understand how to progress through the screens. In addition, to users inputting information to generate a personalized financial plan based on their career metrics.

User Testing

User Testing Prototype

A prototype was designed using the mid fid mobile wireframes. 3 participants were given the task to join On Tour, pick a financial plan and choose an account. User feedback highlights successful completion, but additional direction or instructions would be helpful so users have more clarity to what they are doing.

Design

The UI is a responsive design, created for users to access from both mobile and desktop interfaces.

Conclusion

The UI design of On Tour would benefit from in-depth User centered design research, so features provided are most effective and useful to potential users. This includes consistent research, so the career guidance, strategy and information can change over time.
